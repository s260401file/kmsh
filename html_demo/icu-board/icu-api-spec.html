說明

本規格書由 html_demo/icu-board/ 之 9 個 HTML 頁籤與對應之 js/*Data.js mock JSON 反推產生，用於：

  • 與資訊單位提供之 HIS / TMS API 對照確認可介接欄位
  • 盤點需新建之本地 API 端點
  • 作為後續 ASP.NET Core Controller / Service / Model 開發的依據

ICU 特殊規範：所有欄位採 camelCase（不同於 W52 的 PascalCase），床位 ID 格式為 F4-XX / F3-XX，病患資料欄位以小寫開頭（如 name、doctor）。

病室動態
頁籤 1 · 病室動態 §7.1.2.2 (1) index.html · js/mockData.js 新建端點
GET/api/wards/ICU

用途

取得 ICU 病房基本資訊與 25 床病患明細（4F 20床 + 3F 5床）。同時為「管路」頁籤的資料來源（無需獨立端點）。

.namestringY「高雄市立民生醫院」
.wardstringY「ICU」
.wardDirectorstring—病房主任姓名
.headNursestringY單位護理長姓名
bedsBed[]Y床位陣列（25 筆，含空床）
.idstringY床位 ID，格式 F4-XX / F3-XX
.floorintY樓層：4 或 3
.numintY床號序號（1–22）
.statusenumYoccupied / isolation / transfer / discharge / empty
.patientPatient—病患資料；空床時為 null

Patient 結構（camelCase，ICU 特有欄位標註）

欄位型別必填說明

genderM/FY性別

birthDatestring—出生日期 yyyy/MM/dd

資料來源

  • HIS API：住院病患基本資料（病歷號、姓名、診斷、入院日、主治醫師）
  • 本地 DB：床位配置（4F/3F）、病患旗標（ventilator/crrt/dnr…）、責任護理師指派
  • TMS API：護理長 / 病房主任取自 getOrgUnit.PE_NO_Leader
ICU 特有欄位：ventilator（呼吸器）與 crrt（CRRT）為 ICU 專屬管路旗標，W52 無此欄位。condition 嚴重度分級亦為 ICU 獨有（W52 使用 Condition 但含義略不同）。

抗生素使用清單 ICU 獨有
頁籤 2 · 抗生素 §7.1.2.2 (2) antibiotic.html · js/antibioticData.js ICU 獨有 新建端點
GET/api/wards/ICU/antibiotics

用途

取得 ICU 各床位的抗生素使用明細。UI 以病床平面圖呈現，點選床位後顯示該病人的抗生素清單。

規格書原文：「類似病人詳細資料，不要顯示總表，依照平面圖中的病床，點選病床後顯示該病人的抗生素詳細資訊」

Data 結構

欄位型別必填說明
wardCodestringY「ICU」
queryDatestringY查詢日期 yyyy-MM-dd
bedsBedAbx[]Y所有床位的抗生素資料（25 筆，空床 antibiotics 為空陣列）
.idstringY床位 ID F4-XX / F3-XX
.antibioticsAntibiotic[]Y抗生素清單（無抗生素為空陣列 []）
.antibiotics[].antibioticIdintY抗生素 ID
.antibiotics[].drugNamestringY藥品名稱（含劑量，如 Vancomycin 1g）
.antibiotics[].startDateTimedatetimeY開始時間 yyyy-MM-dd HH:mm
.antibiotics[].firstDoseDateTimedatetimeY首次給藥時間 yyyy-MM-dd HH:mm
.antibiotics[].endDateTimedatetimeY預計結束時間 yyyy-MM-dd HH:mm

手術資訊（前後 3 日共 7 天）
頁籤 4 · 手術資訊 §7.1.2.2 (3) surgery.html · js/surgeryData.js HIS API
GET/api/wards/ICU/surgery-info?date={date}

用途

取得 ICU 病患的手術資訊。規格書要求顯示「當日 + 前後 3 日」共 7 天，UI 以日期切換列呈現。

ICU vs W52 差異：W52 手術資訊僅顯示當日，ICU 需顯示前後 3 日共 7 天（規格書 §7.1.2.2 (3.2)）。

Query 參數

欄位型別必填說明
datestring—查詢日期 yyyy-MM-dd，預設今日。前端可一次請求「今日」，後端回傳前後 3 日所有資料

Data 結構

欄位（camelCase）型別必填說明
wardCodestringY「ICU」
queryDatestringY基準日期
itemsSurgery[]Y手術清單（跨 7 天）
.surgeryIdintY手術 ID
.datestringY手術日期 yyyy-MM-dd（前端依此分組切換）
.orRoomstringY刀房（如 OR-01）
.scheduledTimeHH:mmY排程時間
.bedIdstringY病人所在床位 F4-XX
.patientNamestringY姓名（遮罩）
.genderM/FY性別
.ageintY年齡
.procedurestringY術式（含英文學名）
.diagnosisstringY診斷
.anesthesiaMethodstringY麻醉方式
.attendingSurgeonstringY主治醫師
.statusenumY待手術 / 手術中 / 已完成 / 取消

資料來源

  • HIS API：手術排程系統（OR Scheduling），需包含前後 3 日資料

APPENDIX B
共通回應格式

所有端點採統一包裝（ICU 使用 camelCase 包裝鍵）：

回應包裝 
{
  "success": true,      // bool — 成功 / 失敗
  "message": "",       // string — 錯誤訊息；成功時為空字串
  "data": { ... }      // object | array — 實際資料；失敗時為 null
}
JWT 認證：所有端點需帶 Authorization: Bearer <token> Header。
