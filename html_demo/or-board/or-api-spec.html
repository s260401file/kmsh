說明

本規格書由 html_demo/or-board/ 之 6 個 HTML 頁籤與對應之 js/*Data.js mock JSON 反推產生，用於：

  • 與資訊單位提供之 HIS / TMS API 對照確認可介接欄位
  • 盤點需新建之本地 API 端點
  • 作為後續 ASP.NET Core Controller / Service / Model 開發的依據

OR 欄位命名規範：所有欄位採 PascalCase（與 W52 相同，不同於 ICU 的 camelCase）。刀房 ID 格式為 OR-0X。

手術動態
頁籤 1 · 手術動態 §7.1.2.3 (1) index.html · js/mockData.js 新建端點
GET/api/wards/OR

用途

取得手術室基本資訊與 7 間刀房即時狀態。包含每刀房手術病患資料、手術來源/狀態、護理人員。

Data 結構

欄位（PascalCase）型別必填說明
HospitalInfoobjectY醫院/病房基本資訊
.HospitalNamestringY「高雄市立民生醫院」
.WardCodestringY「OR」
.WardDirectorstring—手術室主任姓名（含職稱）
.HeadNursestringY護理長姓名（含職稱）
RoomsRoom[]Y刀房陣列（7 筆，含空房）
.RoomIdstringY刀房 ID，格式 OR-0X（OR-01 ～ OR-07）
.StatusenumYin-surgery / prep / completed / empty
.Patientobject|null—空房時為 null
.PatientNamestringY病患姓名（匿名化）
.GenderenumYM / F
.AgeintY年齡
.MedRecordstring—病歷號
.BirthDatestring—格式 YYYY/MM/DD
.Departmentstring—科別
.Diagnosisstring—診斷（英文病名）
.SurgeryNamestringY術式名稱（中文）
.SurgerySourceenumY急診刀 / 門診刀 / 住院刀
.SurgeryStatusenumY手術中 / 準備中 / 已完成
.DoctorstringY主刀醫師姓名（含職稱）
.ScrubNursestring—刷手護理師
.CircNursestring—流動護理師
.AnesTypestring—麻醉方式（例：全身麻醉 GA）
.ScheduledTimestring—排程時間，格式 HH:mm
.StartTimestring—手術開始時間，格式 HH:mm
.EndTimestring|null—手術結束時間，進行中為 null
.Notesstring—術前備註
統計欄位：前端統計（刀房總數、手術中、急/門/住院刀、準備中、已完成、空房）由前端依 Status 與 SurgerySource 欄位自行運算，無需後端另計。

Mock 資料摘要（js/mockData.js）

  • OR-01：住院刀，手術中（腹腔鏡膽囊切除術）
  • OR-02：急診刀，手術中（急性盲腸炎）
  • OR-03：急診刀，手術中（股骨骨折）
  • OR-04：住院刀，手術中（心臟瓣膜置換）
  • OR-05：門診刀，準備中（膝關節置換）
  • OR-06：門診刀，已完成（攣縮疤痕植皮）
  • OR-07：空房

手術派班
頁籤 2 · 手術派班 §7.1.2.3 (2) schedule.html · js/scheduleData.js 新建端點
GET/api/wards/OR/schedule?date={date}

用途

取得指定日期三班（白班 / 小夜 / 大夜）的刀房護理師派班、麻醉科人員、體外循環技師。

Query 參數

參數型別必填說明
datestringY查詢日期，格式 YYYY-MM-DD

Data 結構（Shifts[]）

欄位（PascalCase）型別必填說明
ShiftsShift[]Y3 筆（白班 / 小夜 / 大夜）
.ShiftTypeenumY白班 / 小夜 / 大夜
.ShiftTimestringY時段說明（例：08:00–16:00）
.ChargeobjectY值班護理長
.NamestringY護理長姓名
.ExtensionstringY院內分機
.AnesthesiaStaff[]Y麻醉科人員（白班 3 人、小夜 2 人、大夜 1 人）
.NamestringY姓名
.RolestringY職稱（主治/住院醫師 R2/R1 等）
.ExtensionstringY院內分機
.CircTechobject|null—體外循環技師（白班有、小夜/大夜 null）
.RoomsRoomAssign[]Y7 筆，每刀房護理師派班
.RoomIdstringYOR-01 ～ OR-07
.ExtensionstringY刀房分機
.ScrubNursestring|null—刷手護理師（未排班時 null）
.CircNursestring|null—流動護理師（未排班時 null）

欄位（PascalCase）型別必填說明
ItemsHandoverItem[]Y交班項目（空陣列 = 今日無交班）
.HandoverIdintY交班記錄 ID
.RoomIdstringY來源刀房（如 OR-06）
.PatientNamestringY病患姓名（匿名化）
.GenderenumYM / F
.AgeintY年齡
.MedRecordstring—病歷號
.SurgeryNamestringY術式名稱
.SurgerySourceenumY急診刀 / 門診刀 / 住院刀

用途

取得手術室科內公告與院方公告，依優先度（重要優先）排列。資料來源為本地 DB，由護理長透過管理後台維護。

Data 結構

欄位（PascalCase）型別必填說明

HospBulletinsBulletin[]Y院方公告（Mock 3 筆）
.BulletinIdintY公告 ID
.CategorystringY來源分類（如「手術室」「院方」）
.TitlestringY公告標題
.ContentstringY公告內容（自由文字，支援換行 \n）
.PriorityenumY重要 / 一般
.PostedBystringY發布者（含職稱）
.PostedAtstringY發布日期，格式 YYYY-MM-DD
.IsActiveboolY是否顯示（後台可停用）
注意：前端依 Priority === "重要" 排序（重要在前），再依 PostedAt 降序排列。

APPENDIX A
共通回應格式（Wrapper）

所有端點統一使用下列包裝格式：

JSON 範例 
{
  "Success": true,
  "Message": "",
  "Data": {
    "WardCode": "OR",
    "QueryDate": "2026-06-03",
    // ... 各端點資料 ...
  }
}
欄位型別說明
Successbool成功 true，失敗 false
Messagestring錯誤訊息（成功時空字串）
Dataobject各端點回傳資料（失敗時 null）
.WardCodestring病房代碼（OR）
.QueryDatestring查詢日期（YYYY-MM-DD）
APPENDIX B
狀態列舉值

刀房狀態（Room.Status）

值說明卡片樣式
in-surgery手術進行中綠色背景
prep術前準備中黃色背景
completed手術已完成白色背景（略透明）
empty空房（無排刀）灰色背景（虛線框）

手術來源（Patient.SurgerySource）

值說明Badge 顏色
急診刀急診安排手術紅色（#CC3333）
門診刀門診安排手術藍色（#1976D2）
住院刀住院安排手術綠色（#27AE60）

手術狀態（Patient.SurgeryStatus）

值說明
手術中正在進行手術
準備中術前準備階段
已完成手術已結束

公告優先度（Bulletin.Priority）

值說明顏色
重要需優先閱讀紅色色條
一般一般通知綠色色條
