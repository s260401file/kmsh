說明

本規格書由 html_demo/er-board/ 之 6 個 HTML 頁籤與對應之 js/*Data.js mock JSON 反推產生，用於：

  • 與資訊單位提供之 HIS / TMS API 對照確認可介接欄位
  • 盤點需新建之本地 API 端點
  • 作為後續 ASP.NET Core Controller / Service / Model 開發的依據

ER 特殊規範：所有欄位採 PascalCase（同 W52 / OR）。床位以 BedId 識別（如 負1、MER01、MER05）。急診特有 Triage（1–5 分級）與多個布林旗標（Dnr、Isolation、Observation…）。

大量傷患（MCI）無獨立 API：由前端對 /api/wards/ER 回傳的 Beds 資料過濾出非空床，依 Triage 升序排列，不需額外端點。

病室動態
頁籤 1 · 病室動態 §7.1.2.4 (1) index.html · js/mockData.js 新建端點

用途

取得急診室基本資訊與全部床位病患明細（19 床，含空床）。同時為「大量傷患」頁籤的資料來源，無需獨立端點。

ENDPOINT 03
大量傷患（MCI）
頁籤 3 · 大量傷患 §7.1.2.4 (3) mass-casualty.html 無獨立 API
複用/api/wards/ER（ENDPOINT 01）

用途

大量傷患頁面不需獨立 API，由前端對病室動態回傳的 Beds 資料處理：

  • 過濾條件：Status !== "empty" 且 Patient !== null
  • 排序：依 Patient.Triage 升序（1 最危急顯示於首位）
  • 統計：重症（Triage ≤ 2）、輕中症（Triage ≥ 3）、死亡（Deceased === true）、轉出（TransferOut === true）由前端運算
  • 病人註記：顯示 Dnr、Mbd、Aad、Observation、Admitted（含 AdmBedNo）、TransferOut、TransferIn、Deceased
若未來 MCI 事件需額外記錄（啟動時間、事件描述），可另建 POST /api/wards/ER/mci 端點，目前 Demo 階段不需要。

APPENDIX B
狀態與列舉值

床位狀態（Bed.Status）

值說明卡片樣式
occupied看診/留觀中（一般）白底綠邊框
empty空床灰底虛線框
isolation隔離中（一般/負壓）淡紅底紅邊框
observation留觀淡青底青邊框
transfer轉院淡黃底橙邊框
awaiting待床中淡黃底橙邊框

值級別說明顏色
1一級復甦急救（最危急）深紅（#B71C1C）
2二級重症紅（#E05C5C）
3三級緊急橙（#E8A020）
4四級次緊急綠（#2D7A55）
5五級非緊急灰（#607D8B）

檢查狀態（Exam.Status）

值顏色
待執行黃色 badge
執行中藍色 badge
完成綠色 badge

會診狀態（Consult.Status）

值顏色
