互動規則

  • 總床數 / 看診中：僅顯示數字，不可點擊過濾
  • 留觀：過濾 Patient.Observation = true 的床位
  • 待床：過濾 Patient.Awaiting = true 的床位（含一般/加護/隔離）
  • 隔離：過濾 Status = "isolation" 的床位
  • 一二級 危急：過濾 Triage ≤ 2 的床位
  • 三~五級 輕中症：過濾 Triage ≥ 3 的床位
  • DNR：點擊等同 filter bar 點擊「D」病人註記
  • 轉院：過濾 TransferOut 或 TransferIn = true 的床位
  • 再點一次：取消過濾，回到全部顯示
六、圖例 / 過濾列操作
圖例 / 過濾： 死亡 DNR 留觀 轉出 轉入 AAD MBD 住院

操作方式

  • 一~五級按鈕兼作圖例（顏色對應檢傷分級）與過濾器
  • 點擊一級 → 僅顯示 Triage = 1 的床位；二~五級同理
  • 點擊「留觀」→ 僅顯示 Observation = true 的床位
  • 點擊「待床」→ 僅顯示 Awaiting = true 的床位（含三種待床類型）
  • 點擊「隔離」→ 僅顯示 Status = isolation 的床位（含負壓/接觸/飛沫/空氣）
  • 被選中按鈕出現白色光暈邊框；再點一次取消過濾
  • 點擊「全部」強制還原所有床位顯示
  • 右側「夜間版 / 明亮版」按鈕切換配色主題

八、React 串接 API 注意事項

ER 特有串接注意事項

  • BedId 含中文「負1」「負2」，CSS class 轉換邏輯：bedId.replace("負", "neg")
  • 滯留時間由前端動態計算（ArrivalDate + ArrivalTime → 當下時刻差），建議 API 回傳 ISO 8601 timestamp
  • Triage 欄位為 int（1–5），非 string；AwaitingType 可為 null
  • Stats 可由前端從 Beds 陣列即時計算（見 getStats()），或由 API 提供聚合值
  • BedId 命名規則：MER 系列（01–13，跳 04）、OER 系列（01–02）、MER9xx 系列（991–993）
⚠️ MER04 不存在於床位清單中（跳號），後端 API 設計時請確認此間隔是否有特殊用途。
