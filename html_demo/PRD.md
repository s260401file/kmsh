# 產品需求文檔（PRD）
## 高雄民生醫院 ICU 電子白板系統

- **產品名稱**：民生醫院 ICU 電子白板
- **版本**：v1.2
- **文檔日期**：2026-05-22（v1.2 更新：2026-05-22）
- **產品定位**：供高雄民生醫院護理站大螢幕即時顯示 ICU 雙樓層病床狀態與病患資訊的電子白板系統

---

## 1. 產品概述

取代傳統紙本白板，讓護理站人員在 55 吋大螢幕上（觀看距離約 3 公尺），快速掌握 ICU 全區 25 床的即時病況，包含病患基本資料、醫護分配、床位狀態、醫療設備使用情形等關鍵資訊。

---

## 2. 目標用戶

- **用戶畫像**：醫院 ICU 護理站護理師、護理長、病房主任等院方人員
- **使用場景**：
  - 護理站輪班交接時快速掌握全區狀況
  - 查看各床位病患資訊與醫療狀態
  - 遠距離（約 3 公尺）觀看 55 吋電視螢幕
- **用戶痛點**：
  - 紙本白板資訊更新不即時，易有誤差
  - 距離遠導致字體太小難以閱讀
  - 無法快速篩選特定狀態病床（如隔離、DNR）

## 5. 詳情視窗（Modal）— 欄位說明

點擊床卡後彈出的詳細資訊，包含以下欄位：

- **頂部**：床號、病患姓名（M藍/F粉）、性別/年齡、全部 Badge 標記
- **第一列**：診斷（Diagnosis）— 全寬
- **第二列**：病歷號 / 生日（YYYY/MM/DD）/ 科別
- **第三列**：主治醫師 / 責任護理師
- **第四列**：入院日期 / 住院天數 / 病況等級（穩定/重症/危急）
- **第五列**：隔離狀態 / DNR
- **第六列**：呼吸器 / CRRT
- **第七列**：備註 — 全寬

## 6. 用戶流程

```
護理師進入頁面
    → 看到頁首（ICU / 病房主任 / 護理長 / 時間）
    → 看到跑馬燈（院內公告）
    → 看到圖例/過濾列（13種 Badge 彩色按鈕）
    → 主畫面：左區 4F（20床）/ 右區 3F（5床）+ 統計面板
        → 依顏色快速辨識床位狀態
        → 直接點擊統計數字（如「重症 15」）→ 僅顯示重症床
        → 再點一次統計數字 → 還原全部
        → 或點擊圖例列 Badge 按鈕 → 篩選對應標記床位
        → 點擊特定床卡 → 彈出詳情 Modal 查看完整資料
    → 點擊底部頁籤切換其他功能（抗生素、派班等）
```

---

## 7. UI/UX 設計要求

- **字體大小**：床號 ≥ 22px、病患姓名 ≥ 18px、跑馬燈 ≥ 20px、底部頁籤 ≥ 24px；統計數字 46px，確保 3 公尺距離可閱讀
- **性別顏色**：男性（M）姓名顯示藍色、女性（F）顯示粉紅色（夜間/明亮各有專屬深淺版本）
- **高對比度**：頁首文字統一使用 `--text-stats`（白色系）確保在深色/深綠背景皆清晰；頁籤 active 使用白底綠字最大化辨識度
- **色系**：夜間版（預設）深海軍藍底白字；明亮版民生綠底白字；支援即時切換並儲存偏好
- **螢幕解析度**：針對 55 吋 1920×1080 螢幕優化
- **佈局比例**：4F 區域（flex:4）約 57%，3F 區域（flex:3）約 43%（含統計面板）；3F 卡片與 4F 等高

---

## 8. 非功能需求

- **性能要求**：頁面載入 < 3 秒，資料更新間隔 ≤ 60 秒
- **兼容性**：支援 Chrome、Edge 最新版本（護理站常用瀏覽器）
- **安全性**：病患資料顯示需符合個資保護原則，正式版需身份驗證機制
- **資料來源**：v1.x 使用 mock 示意資料，v2.0 串接 HIS API
