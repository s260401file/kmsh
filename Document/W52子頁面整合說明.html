W52 子頁面整合說明

說明 W52-board html_demo 的 11 個頁面如何整合進 React 巢狀路由
適用版本：React 19 + React Router 7  |  最後更新：2026-06-03

整合目標

html_demo W52-board 設計稿共有 11 個 HTML 頁面，底部 Tab 已改為 <a href="*.html"> 超連結導覽。目標是將這 11 個頁面整合進 React，使用 React Router 巢狀路由實作，讓 Tab 點擊時透過 URL 路由切換，不需重載頁面。

💡
設計稿是唯一真實來源
所有視覺設計以 html_demo/W52-board/ 為準。設計師更新 html_demo → 通知開發者同步到 React。

路由結構

使用 React Router 的 巢狀路由（Nested Routes）+ Outlet 模式：

/w52 → W52Layout（共用 header + tabs）
/w52 → redirect → /w52/ward
/w52/ward → WardTab（病室動態）
/w52/care → CareTab（照護提醒）
/w52/surgery → SurgeryTab（手術資訊）
/w52/exam → ExamTab（檢查/會診）
/w52/contact → ContactTab（連絡電話）
/w52/schedule → ScheduleTab（排班資訊）
/w52/doctor → DoctorTab（醫師資訊）
/w52/bulletin → BulletinTab（佈告欄）
/w52/evacuation → EvacuationTab（避難圖）
/w52/handover → HandoverTab（護理交班）
/w52/team → TeamTab（照護團隊）

App.jsx 中的寫法：

// App.jsx — W52 巢狀路由
<Route path="/w52" element={<W52Layout />}>
  <Route index element={<Navigate to="ward" replace />} />
  <Route path="ward"       element={<WardTab />} />
  <Route path="care"       element={<CareTab />} />
  <Route path="surgery"    element={<SurgeryTab />} />
  <Route path="exam"       element={<ExamTab />} />
  <Route path="contact"    element={<ContactTab />} />
  <Route path="schedule"   element={<ScheduleTab />} />
  <Route path="doctor"     element={<DoctorTab />} />
  <Route path="bulletin"   element={<BulletinTab />} />
  <Route path="evacuation" element={<EvacuationTab />} />
  <Route path="handover"   element={<HandoverTab />} />
  <Route path="team"       element={<TeamTab />} />
</Route>

關鍵設計：各 Tab 渲染責任

Tab渲染什麼說明
WardTab <main> + filter-bar + modal 用 Fragment <>...</> 返回多個元素，讓 filter-bar 出現在 Layout <Outlet> 的位置，保留原始 CSS 結構
其他 9 個 Tab <main className="main-content"> 包含各自面板 無 filter-bar，結構簡單

NavLink 取代 <button>

底部 Tab 改用 NavLink，自動偵測當前路由並加 active class：

// 舊：button 切換 activeTab state（無 URL 變化）
<button className={`tab-btn${activeTab===i?' active':''}`} onClick={()=>setActiveTab(i)}>

// 新：NavLink 自動處理 active 狀態
<NavLink
  to="care"
  className={({ isActive }) => `tab-btn${isActive ? ' active' : ''}`}
>
  照護提醒
</NavLink>
⚠️
NavLink 渲染為 <a> 標籤，CSS 需補上 text-decoration: none， 已在 W52Layout.css 的 .w52-board .tab-btn 規則中加入。

CSS 轉換規則

每個 Tab 有自己的 CSS 檔（tabsCss/*.css），轉換時所有選擇器加 .w52-board 前綴以避免樣式衝突：

html_demo CSS 寫法React tabsCss 寫法
.care-panel { ... }.w52-board .care-panel { ... }
.care-item.priority-bar-高 { ... }.w52-board .care-item.priority-bar-高 { ... }
各 Tab 的 CSS 變數引用（var(--accent-green)）由 W52Layout.css 的 .w52-board { ... } 提供，不需重複定義

資料層說明

html_demo 的每個 *Data.js 都以 PascalCase 欄位定義 Mock 資料，對應後端 C# Model。React 版本直接複製，只加一行 export default：

// html_demo/js/careData.js
const _MOCK_CARE_REMINDERS = { "Data": { "Items": [...] } }
async function getCareReminders(wardCode) { ... }

// React tabsData/careData.js（只加最後一行）
const CARE_DATA = { "Data": { "Items": [...] } }
export default CARE_DATA

日後換成真實 API

每個 Tab 目前直接 import 靜態資料：

// 現在（Mock）
import CARE_DATA from '../tabsData/careData'
const items = CARE_DATA.Data.Items

// 日後（API）
const [items, setItems] = useState([])
useEffect(() => {
  fetch(`/api/wards/W52/care-reminders`)
    .then(r => r.json())
    .then(data => setItems(data.Data.Items))
}, [])
💡
API 欄位結構與 Mock 完全相同（PascalCase），換 API 時不需修改 JSX 的欄位存取邏輯。

JSX 轉換模式

每個 Tab 的 HTML → JSX 轉換遵循統一模式：

html_demo HTMLReact JSX
class="care-panel"className="care-panel"
id="care-list"（JS 注入內容）{items.map(item => <div key={item.ReminderId}>...</div>)}
stroke-width="2"（SVG 屬性）strokeWidth="2"（camelCase）
JS 動態計算的日期格式保留在元件 function 內，直接計算後放入 JSX
onclick 事件（班別切換）useState + onClick

特殊處理：排班 Tab 班別切換

// ScheduleTab.jsx — 用 useState 取代 JS 全局 currentShift
const [activeIdx, setActiveIdx] = useState(0)
const shift = shifts[activeIdx]

<button
  className={`sc-shift-btn${i === 0 ? ' is-current' : ''}${activeIdx === i ? ' active' : ''}`}

特殊處理：避難圖 SVG

HTML 的 <svg> 直接搬入 JSX，只需把 kebab-case 屬性改為 camelCase：

// HTML：stroke-width="3.5" stroke-dasharray="10,5"
// JSX：
strokeWidth="3.5" strokeDasharray="10,5"

日後更新流程

設計師更新 html_demo 後，依照以下步驟同步到 React：

情況一：某個 Tab 的 CSS 有改動

找到對應的 tabsCss/*.css，用新的 CSS 覆蓋，並在所有選擇器前補上 .w52-board。

情況二：某個 Tab 的 HTML 結構有改動

比對 html_demo/*.html 的 <main> 區塊與 tabs/*.jsx 的差異，依 HTML → JSX 轉換規則更新。

情況三：資料欄位有新增

更新 tabsData/*.js 的資料結構，同步更新 tabs/*.jsx 中對應的渲染邏輯。

情況四：新增 Tab 頁面

  1. 在 html_demo 設計新的 *.html 和對應 CSS
  2. 在 React 建立 tabsData/newData.js（複製 + export default）
  3. 在 tabsCss/new.css（複製 + .w52-board 前綴）
  4. 建立 tabs/NewTab.jsx（HTML → JSX 轉換）
  5. 在 W52Layout.jsx 的 TABS 陣列加入新 Tab
  6. 在 App.jsx 加入新路由 <Route path="new" element={<NewTab />} />
