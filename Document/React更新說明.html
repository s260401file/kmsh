護理白板系統 — React 前端更新說明

每次修改 React 程式碼後，需重新 Build 並部署到 IIS 才會生效  |  最後更新：2026-05-30

快速更新（兩個指令）

在開發機上開啟 PowerShell，執行以下兩行：

# 第一步：進入 React 專案目錄並 Build
cd "C:\WorkDir\Nursing Whiteboard\React"
npm run build

# 第二步：複製 build 產出到 IIS 目錄
Copy-Item -Path ".\dist\*" -Destination "C:\inetpub\wwwroot\kmsh-whiteboard" -Recurse -Force
✅
完成後重新整理瀏覽器（Ctrl + F5）即可看到最新版本，不需重啟 IIS。

詳細步驟說明

1
完成程式碼修改
在 C:\WorkDir\Nursing Whiteboard\React\src\ 下修改或新增頁面元件。
2
執行 Build
cd "C:\WorkDir\Nursing Whiteboard\React"
npm run build
Build 完成後會在 React\dist\ 產出靜態檔案（index.html、assets/、web.config）。
成功時終端機會顯示類似： 
✓ built in 580ms
3
複製到 IIS 目錄
Copy-Item -Path ".\dist\*" -Destination "C:\inetpub\wwwroot\kmsh-whiteboard" -Recurse -Force
此指令會覆蓋舊版本的所有檔案。
4
瀏覽器強制重新整理
按 Ctrl + F5（或 Ctrl + Shift + R）強制清除快取並重新載入頁面。

新增頁面的標準流程

若本次更新包含新增路由頁面，需同時修改以下兩個檔案：

① 新增頁面元件

在 src/pages/ 建立新檔案，例如 NewPage.jsx：

import { Link } from 'react-router-dom'

function NewPage() {
  return (
    <div>
      <p>頁面內容</p>
      <Link to="/">返回首頁</Link>
    </div>
  )
}

export default NewPage

② 在 App.jsx 註冊路由

開啟 src/App.jsx，加入 import 和 Route：

import NewPage from './pages/NewPage'

// Routes 區塊內加入：
<Route path="/new" element={<NewPage />} />

③ 在首頁加入按鈕

開啟 src/pages/Home.jsx，加入按鈕：

<button onClick={() => navigate('/new')}>新頁面</button>

重要目錄對照

用途路徑
React 原始碼 C:\WorkDir\Nursing Whiteboard\React\src\
Build 產出（中繼） C:\WorkDir\Nursing Whiteboard\React\dist\
IIS 前端部署目錄 C:\inetpub\wwwroot\kmsh-whiteboard\
IIS web.config 來源 C:\WorkDir\Nursing Whiteboard\React\public\web.config
Build 時自動複製進 dist，不需手動處理

注意事項

⚠️
不要直接修改 IIS 目錄的檔案
C:\inetpub\wwwroot\kmsh-whiteboard\ 裡的檔案每次 Build 後都會被覆蓋， 請一律修改 src\ 原始碼再重新 Build。
💡
開發環境測試
修改程式碼時可先在開發模式即時預覽，確認無誤後再 Build 部署： 
cd "C:\WorkDir\Nursing Whiteboard\React"
npm run dev
開啟 http://localhost:5173 即可看到即時更新（修改儲存後自動刷新）。
💡
API 未同步更新的情況
若同時也修改了 .NET API，需另外執行 API 的更新流程， 詳見《IIS 部署說明.html》的「日後更新方式」章節。
