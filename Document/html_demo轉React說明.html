html_demo 轉 React 說明

說明 html_demo/ 設計稿如何對應到 React 元件，以及日後更新設計稿的同步流程
最後更新：2026-05-30

設計稿對應關係

每個 html_demo 資料夾對應一個 React 板，四者一一對應：

html_demo 資料夾React 路由React 元件路徑CSS scope class
html_demo/W52-board//w52pages/boards/W52/W52Page.jsx.w52-board
html_demo/icu-board//icupages/boards/ICU/IcuPage.jsx.icu-board
html_demo/or-board//orpages/boards/OR/OrPage.jsx.or-board
html_demo/er-board//erpages/boards/ER/ErPage.jsx.er-board
💡
設計稿是唯一真實來源（Source of Truth）
所有視覺設計以 html_demo/ 為準。設計師更新 html_demo → 通知開發者同步到 React。 React 程式碼不應獨立修改樣式，避免兩邊不一致。

CSS 轉換規則

html_demo 各板的 CSS class 名稱相同（如 .page-header、.bed-card），若直接匯入會互相覆蓋。解法：所有 CSS 選擇器加板專屬前綴。

合併 4 個 CSS 檔案 → 1 個

html_demo（4個檔案）
css/variables.css   ← :root { --bg-header: ... }
css/reset.css       ← body { display: flex; }
css/main.css        ← .page-header { ... }
css/components.css  ← .bed-card { ... }
→
React（1個檔案）
W52Page.css         ← 4個合併，全部加 .w52-board 前綴

三種轉換模式

html_demo 寫法React CSS 寫法說明
:root { --bg-header: #2D7A55; } .w52-board { --bg-header: #2D7A55; } CSS 變數改為板作用域
body { display: flex; } .w52-board { display: flex; } body 樣式改為 wrapper div
.page-header { background: ... } .w52-board .page-header { background: ... } 所有選擇器加板前綴
⚠️
更新 CSS 時，只需把 html_demo 的 CSS 合併後，在每個選擇器前加上對應的前綴（.w52-board、.icu-board 等）即可。 工具或編輯器的「全域取代」功能可快速完成。

JS 邏輯轉換

html_demo 的 JS 邏輯轉換對應如下：

html_demo 機制React 對應位置
setInterval(updateClock, 1000)
操作 DOM #clock-date		 useClock() hook
回傳 { date, time } state		 src/hooks/useClock.js（共用）
let currentFilter = "all"
全域變數		 const [filter, setFilter] = useState('all') 各板 JSX 元件內
applyFilter()
操作 DOM classList		 isBedVisible(bed, filter)
純函數，回傳 boolean		 各板 JSX 元件內
openModal(bed)
填入 30+ 個 #m-* DOM		 const [selectedBed, setSelectedBed] = useState(null)
{selectedBed && <BedModal bed={selectedBed} />}		 各板 JSX 元件內
renderBedCard(bed)
回傳 HTML template string		 <BedCard bed={bed} />
React 元件		 各板 JSX 元件內
MOCK_DATA.Beds.forEach()
手動插入 DOM		 MOCK_DATA.Beds.map(bed => <BedCard />) JSX 內直接渲染

範例：Filter 邏輯

html_demo（main.js）
let currentFilter = "all";
function setFilter(filter) {
  currentFilter = filter;
  document.querySelectorAll(".bed-card")
    .forEach(card => {
      let show = false;
      switch (currentFilter) {
        case "surgery": show = card.dataset.surgery==="1"; break;
        ...
      }
      card.classList.toggle("filtered-out", !show);
    });
}
→
React（JSX）
const [filter, setFilter] = useState('all');

function isBedVisible(bed, filter) {
  if (filter === 'all') return true;
  switch (filter) {
    case 'surgery': return !!bed.Patient?.Surgery;
    ...
  }
}

// 渲染時傳入 filteredOut prop
{MOCK_DATA.Beds.map(bed => (
  <BedCard
    key={bed.BedId}
    bed={bed}
    filteredOut={!isBedVisible(bed, filter)}
  />
))}

HTML → JSX 轉換規則

HTML 結構幾乎原封不動搬入 JSX，只有以下機械性替換：

HTML 寫法JSX 寫法
class="page-header"className="page-header"
onclick="setFilter('surgery')"onClick={() => handleFilter('surgery')}
<div id="bedModal"> + display toggle{selectedBed && <BedModal />}
id="clock-date"（DOM 更新）{date}（state 直接渲染）
id="stat-total"（DOM 更新）{stats.total}（state 直接渲染）
template string 動態生成 HTMLJSX .map() 迴圈
style="grid-column:1/5"style={{gridColumn:'1/5'}}
HTML 自閉合標籤 <div/>（可選）JSX 必須自閉合 <div />

日後更新工作流程

設計師更新 html_demo 後，對應修改 React 的步驟：

情況一：只有 CSS 樣式變動

1
確認哪個板的哪個 CSS 檔案有變動（variables / main / components）
2
開啟對應的 React CSS 檔（如 pages/boards/W52/W52Page.css），找到對應規則修改
注意：React CSS 的選擇器有 .w52-board 前綴，不要忘記加
3
執行 npm run dev 預覽確認，再 build 部署

情況二：HTML 結構有變動

1
比對 html_demo 的 index.html body 與 React JSX 的差異
2
依照「HTML → JSX 轉換規則」更新對應板的 *Page.jsx 中的 JSX 結構
（class → className、inline onclick → onClick 等機械替換）
3
若有新增 data-* 屬性對應新的 filter 邏輯，同步更新 isBedVisible() 函數

情況三：JS 邏輯有變動

1
確認 main.js 或 components.js 的變動內容
2
對應更新 React 元件：
• buildBadges() 改動 → 更新 JSX 內同名函數
• applyFilter() 改動 → 更新 isBedVisible()
• openModal() 改動 → 更新 *Modal 元件的 JSX

情況四：mockData 資料結構有變動

1
複製更新後的 mockData.js 到對應 React 資料夾
2
在最後加上 export default MOCK_DATA; 和 export function getStats
3
若欄位名稱有變動，同步更新 JSX 中存取的屬性名稱（如 bed.Patient.PatientName）

部署到 IIS（每次更新後執行）

# 進入 React 專案目錄
cd "C:\WorkDir\Nursing Whiteboard\React"

# Build 生產版本
npm run build

# 複製到 IIS 目錄
Copy-Item -Path ".\dist\*" -Destination "C:\inetpub\wwwroot\kmsh-whiteboard" -Recurse -Force
✅
部署完成後重新整理瀏覽器（Ctrl + F5）即可，不需重啟 IIS。

各板特殊差異說明

板特殊之處
W52 資料欄位採 PascalCase（PatientName、AttendingDoctor）
床位 ID 格式：W52-001，顯示時去掉前綴
ICU 資料欄位採 camelCase（name、doctor）
雙樓層結構（4F / 3F），統計面板放在 3F section 內
有額外欄位：ventilator、crrt、ng
OR 資料結構是 Rooms（非 Beds），卡片 class 是 or-card（非 bed-card）
Modal 有手術時長計算（開始/結束時間差）
來源分類：急診刀 / 門診刀 / 住院刀
ER 床位 ID 含中文字「負」→ CSS class 需轉換：負2 → bed-neg2
檢傷分級 1-5 級（triage badge）
有 nursing-station（護理站）元素
Modal 計算到院滯留時間
